Feature Highlight: Bumped Emissive Sprite Shader

struct Input
{
	float2 uv_MainTex;
	fixed4 color;
	fixed4 emissionTint;
};

void vert (inout appdata_full v, out Input o)
{
	#if defined(PIXELSNAP_ON) && !defined(SHADER_API_FLASH)
	v.vertex = UnityPixelSnap (v.vertex);
	#endif
	v.normal = float3(0,0,-1);
	v.tangent =  float4(1, 0, 0, 1);
	
	UNITY_INITIALIZE_OUTPUT(Input, o);
	o.color = _Color;
}

void surf (Input IN, inout SurfaceOutput o)
{
	fixed4 c = tex2D(_MainTex, IN.uv_MainTex) * IN.color;
	o.Albedo = c.rgb;
	o.Alpha = c.a;
	o.Normal = UnpackNormal(tex2D(_BumpMap, IN.uv_MainTex));
	o.Emission = c.rgb * tex2D(_MainTex, IN.uv_MainTex) * tex2D(_Emission, IN.uv_MainTex).r * _EmissionTint;
}

To create a distinct visual style, improving our ability to stand out in a very saturated market, I implemented a bumped sprite shader (with support for emission maps aswell).
The shader requires an albedo map (the original sprite, used for the diffuse), with options for a standard normal map - and a grayscale emission map, where white indicates light.
It also features some optional parameters such as albedo and emissive color tints (emissive tint supports HDR colors, which looks excellent in combination with post-processing).
Sadly, due to targeting older versions of android hardware this shader did not make it into the final release.

DPAD Controls

Bandana Bones user interface, landscape mode on a LG Nexus 5X.

Growing up in the later 90's, I have fond memories of old-school DPAD-controls on early GameBoy handheld consoles.
This inspired the user interface in Bandana Bones - movement feels very natural and is incredibly easy to understand.
The same applies to interactions, which appear/disappear depending on if the player character is facing an interactable object, together with a header.
An interesting point in development was guaranteeing that the UI displayed properly in both portrait/landscape mode across a large range of devices with varying screen sizes/resolutions.
